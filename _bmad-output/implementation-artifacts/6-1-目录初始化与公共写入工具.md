### `common/doris_writer.py` 实现规范

**函数签名**：
```python
import logging
import pymysql
from typing import Optional

logger = logging.getLogger(__name__)

def write_to_doris(
    table: str,
    records: list[dict],
    unique_keys: list[str],
    source: str = "unknown",
    batch_size: int = 1000,
) -> int:
    """
    upsert 写入 Doris 表。

    Args:
        table: Doris 表名（含 database 前缀，如 "hqware.tw_pixel_orders"）
        records: 待写入数据，list[dict]，每个 dict 的 key 为列名
        unique_keys: 主键列名列表，用于构造 ON DUPLICATE KEY UPDATE
        source: 日志标识符（如 "triplewhale"）
        batch_size: 每批写入行数，默认 1000

    Returns:
        成功写入总行数

    Raises:
        RuntimeError: 写入失败时抛出，不静默
    """
```

**实现要点**：

```python
# 1. 获取连接
from doris_config import DorisConfig
config = DorisConfig()
conn = pymysql.connect(**config.DB_CONFIG)

# 2. 写入前必须执行的 SET 语句（顺序不可变）
cursor.execute("SET enable_unique_key_partial_update = true")
cursor.execute("SET enable_insert_strict = false")

# 3. 构造 upsert SQL（Doris 使用 INSERT INTO ... ON DUPLICATE KEY UPDATE）
#    列名和 VALUES 均需动态构造，禁止字符串拼接（使用 %s 占位）
columns = list(records[0].keys())
placeholders = ", ".join(["%s"] * len(columns))
update_clause = ", ".join(
    [f"{col} = VALUES({col})" for col in columns if col not in unique_keys]
)
sql = f"INSERT INTO {table} ({', '.join(columns)}) VALUES ({placeholders}) ON DUPLICATE KEY UPDATE {update_clause}"

# 4. executemany 分批写入
for i in range(0, len(records), batch_size):
    batch = records[i:i + batch_size]
    values = [tuple(r[col] for col in columns) for r in batch]
    cursor.executemany(sql, values)
    conn.commit()

# 5. 日志格式
logger.info(f"[{source}][{table}] 写入 {len(records)} 行 ... 成功")
```

**错误处理规范**：
- 连接失败、SQL 执行失败：抛出 `RuntimeError(f"[{source}][{table}] 写入失败：{e}")`
- 禁止静默失败（`except Exception: pass`）
- `conn.close()` 必须在 `finally` 块中执行

### 单元测试规范

**测试文件位置**：`tests/test_doris_writer.py`（主仓库 tests/ 目录，而非 bi/ 内）

**测试策略**：mock `pymysql.connect`，不依赖真实 Doris 连接。

```python
from unittest.mock import patch, MagicMock

@patch("pymysql.connect")
def test_write_to_doris_basic(mock_connect):
    mock_conn = MagicMock()
    mock_cursor = MagicMock()
    mock_connect.return_value = mock_conn
    mock_conn.cursor.return_value.__enter__ = lambda s: mock_cursor
    mock_conn.cursor.return_value.__exit__ = MagicMock(return_value=False)

    # 验证 executemany 被调用
    # 验证 SET 语句被执行
    # 验证连接被关闭
```

**关键测试场景**：
1. 正常写入：records 非空，executemany 被调用，commit 被调用
2. 批量分页：records 数量 > batch_size，验证分批调用次数
3. SET 语句：验证 `enable_unique_key_partial_update = true` 在写入前执行
4. 空 records：应提前返回，不创建连接
5. 连接失败：pymysql.connect 抛出异常 → write_to_doris 抛出 RuntimeError
