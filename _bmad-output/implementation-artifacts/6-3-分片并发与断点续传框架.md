# Story 6.3：分片并发与断点续传框架

Status: done

## Story

作为数据工程师，
我希望大数据量表（如 TripleWhale sessions_table）的全量拉取支持分片并发和断点续传，
以便避免单次长时间拉取超时或中断后需从头重来。

## Acceptance Criteria

1. 新建 `hqware.etl_chunk_status` 表（Unique Key 模型），字段：`source`、`table_name`、`chunk_start`、`chunk_end`、`chunk_status`（pending/done/failed）、`created_at`、`updated_at`；提供 `ensure_chunk_table()` 幂等建表函数
2. 全量拉取时按时间区间自动分片，分片粒度可配置（默认 `chunk_days=30`）
3. 支持并发执行多个分片，并发数可配置（默认 `workers=4`）
4. 重启时自动跳过 `done` 状态分片，仅重跑 `pending` 和 `failed` 分片
5. 单片失败不影响其他分片继续执行，失败分片状态记为 `failed` 并打印完整错误
6. 提供通用接口：`chunked_fetch(source, table, fetch_fn, start, end, chunk_days, workers)`
7. 每个分片的 `chunk_status` 在执行前设为 `pending`，成功后设为 `done`，失败后设为 `failed`
8. `tests/test_chunk_runner.py` 中全部单元测试通过（mock pymysql + mock concurrent）

### 为什么新建 `etl_chunk_status` 而非扩展 `etl_watermark`

Story spec 说"etl_watermark 表扩展分片状态字段"，但 `etl_watermark` 当前 UNIQUE KEY 为 `(source, table_name)`，每个 source+table 只有一条记录；分片需要每个 chunk 一条记录，无法共用同一 UNIQUE KEY。

**实现决策**：新建独立的 `etl_chunk_status` 表（UNIQUE KEY = `(source, table_name, chunk_start, chunk_end)`），职责分离：
- `etl_watermark`：记录整体水位线（上次成功时间），Story 6.2 成果，不修改
- `etl_chunk_status`：记录每个时间片的执行状态，本 Story 新增

```sql
CREATE TABLE IF NOT EXISTS hqware.etl_chunk_status (
    source       VARCHAR(64)   NOT NULL COMMENT '数据源标识，如 triplewhale',
    table_name   VARCHAR(128)  NOT NULL COMMENT '表名，如 sessions_table',
    chunk_start  DATETIME      NOT NULL COMMENT '分片起始时间（含）',

    updated_at   DATETIME      NOT NULL COMMENT '最后更新时间（UTC）'
)
UNIQUE KEY(source, table_name, chunk_start, chunk_end)
DISTRIBUTED BY HASH(source) BUCKETS 4
PROPERTIES (
    "replication_num" = "1"
);
```

#### 函数 2：`_generate_chunks(start, end, chunk_days) -> list[tuple[datetime, datetime]]`

```python
def _generate_chunks(
    start: datetime, end: datetime, chunk_days: int
) -> list[tuple[datetime, datetime]]:
    """
    将 [start, end) 按 chunk_days 切分为等长分片。
    最后一片不足 chunk_days 时取到 end。
    """
    from datetime import timedelta
    chunks = []
    current = start
    delta = timedelta(days=chunk_days)
    while current < end:
        next_dt = min(current + delta, end)
        chunks.append((current, next_dt))
        current = next_dt
    return chunks
```

#### 函数 3：`_get_done_chunks(source, table) -> set[tuple]`

从 Doris 查出该 source+table 已经是 `done` 的所有 (chunk_start, chunk_end) 集合，用于跳过。

```python
def _get_done_chunks(source: str, table: str) -> set[tuple]:
    """返回 (chunk_start, chunk_end) 集合，状态为 done 的分片。"""
    config = DorisConfig()
    conn = None
    try:
        conn = pymysql.connect(**config.DB_CONFIG)
        with conn.cursor(pymysql.cursors.DictCursor) as cursor:
            cursor.execute(
                "SELECT chunk_start, chunk_end FROM hqware.etl_chunk_status "
                "WHERE source=%s AND table_name=%s AND chunk_status='done'",
                (source, table),
            )
            rows = cursor.fetchall()
        return {(r["chunk_start"], r["chunk_end"]) for r in rows}
    except Exception as e:
        raise RuntimeError(f"[chunk_runner][{source}/{table}] 查询 done 分片失败：{e}") from e
    finally:
        if conn:
            conn.close()
```

#### 函数 4：`_set_chunk_status(source, table, chunk_start, chunk_end, status) -> None`

使用 `write_to_doris` 执行 upsert：

```python
def _set_chunk_status(
    source: str,
    table: str,
    chunk_start: datetime,
    chunk_end: datetime,
    status: str,
) -> None:
    """upsert 更新分片状态。"""
    now = datetime.utcnow()
    record = {
        "source": source,
        "table_name": table,
        "chunk_start": chunk_start,
        "chunk_end": chunk_end,
        "chunk_status": status,
        "created_at": now,
        "updated_at": now,
    }
    write_to_doris(
        table=CHUNK_STATUS_TABLE,
        records=[record],
        unique_keys=["source", "table_name", "chunk_start", "chunk_end"],
        source=_CHUNK_SOURCE,
    )
```

#### 函数 5：`chunked_fetch(source, table, fetch_fn, start, end, chunk_days, workers)` — 主入口

```python
from concurrent.futures import ThreadPoolExecutor, as_completed
from typing import Callable

def chunked_fetch(
    source: str,
    table: str,
    fetch_fn: Callable[[datetime, datetime], None],
    start: datetime,
    end: datetime,
    chunk_days: int = 30,
    workers: int = 4,
) -> None:
    """
    按时间区间分片并发拉取，支持断点续传。

    Args:
        source: 数据源标识（如 "triplewhale"）
        table: 表名（如 "sessions_table"）
        fetch_fn: 可调用对象，签名 fetch_fn(chunk_start, chunk_end) -> None
                  负责实际数据拉取和写入 Doris；失败时抛出异常
        start: 全量拉取起始时间
        end: 全量拉取结束时间
        chunk_days: 每片时间长度（天），默认 30
        workers: 最大并发线程数，默认 4
    """
    ensure_chunk_table()

    all_chunks = _generate_chunks(start, end, chunk_days)
    done_chunks = _get_done_chunks(source, table)

    pending = [
        (cs, ce) for (cs, ce) in all_chunks
        if (cs, ce) not in done_chunks
    ]

    logger.info(
        f"[chunk_runner][{source}/{table}] "
        f"共 {len(all_chunks)} 片，跳过 {len(all_chunks) - len(pending)} 片（done），"
        f"待执行 {len(pending)} 片，并发数={workers}"
    )

    if not pending:
        logger.info(f"[chunk_runner][{source}/{table}] 所有分片已完成，无需重跑")
        return

    failed_count = 0

    def _run_chunk(chunk_start: datetime, chunk_end: datetime) -> tuple[datetime, datetime, bool, str]:
        _set_chunk_status(source, table, chunk_start, chunk_end, "pending")
        try:
            fetch_fn(chunk_start, chunk_end)
            _set_chunk_status(source, table, chunk_start, chunk_end, "done")
            logger.info(f"[chunk_runner][{source}/{table}] 分片 {chunk_start}~{chunk_end} ... 成功")
            return chunk_start, chunk_end, True, ""
        except Exception as e:
            _set_chunk_status(source, table, chunk_start, chunk_end, "failed")
            error_msg = str(e)
            logger.error(
                f"[chunk_runner][{source}/{table}] 分片 {chunk_start}~{chunk_end} ... 失败：{error_msg}"
            )
            return chunk_start, chunk_end, False, error_msg

    with ThreadPoolExecutor(max_workers=workers) as executor:
        futures = {
            executor.submit(_run_chunk, cs, ce): (cs, ce)
            for (cs, ce) in pending
        }
        for future in as_completed(futures):
            _, _, success, _ = future.result()
            if not success:
                failed_count += 1

    if failed_count > 0:
        raise RuntimeError(
            f"[chunk_runner][{source}/{table}] {failed_count} 个分片失败，详见上方日志"
        )
    logger.info(f"[chunk_runner][{source}/{table}] 全部 {len(pending)} 片成功完成")
```
