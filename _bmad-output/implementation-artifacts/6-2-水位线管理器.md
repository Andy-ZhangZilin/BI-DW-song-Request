# Story 6.2：水位线管理器

Status: done

## Story

作为数据工程师，
我希望有一套水位线管理机制，自动判断首次全量还是后续增量，
以便采集脚本无需手动区分运行模式。

## Acceptance Criteria

1. 在 Doris 中创建 `hqware.etl_watermark` 表，Unique Key 模型，字段：`source`、`table_name`、`last_success_time`、`run_mode`、`updated_at`
2. 提供 `get_watermark(source, table)` → 返回水位线记录（dict）或 `None`（首次运行）
3. 提供 `update_watermark(source, table, success_time)` → 仅在成功写入 Doris 后调用，更新 `last_success_time` 和 `updated_at`
4. 提供 `reset_watermark(source, table)` → 删除该 source/table 的水位线记录，由 `--mode full` 参数触发
5. 首次运行（`get_watermark` 返回 None）→ 调用方触发全量拉取（watermark 模块本身不决策采集逻辑）
6. 后续运行 → `get_watermark` 返回含 `last_success_time` 的记录，调用方用此值作为 `start_date` 做增量
7. `update_watermark` 只在写入 Doris 成功后调用，失败时不调用（调用方职责，watermark 模块本身不检测写入结果）
8. 提供 `ensure_table()` 函数，在首次使用时自动建表（幂等，已存在不报错）

### Doris 建表 DDL（精确）

```sql
CREATE TABLE IF NOT EXISTS hqware.etl_watermark (
    source        VARCHAR(64)  NOT NULL COMMENT '数据源标识，如 triplewhale',
    table_name    VARCHAR(128) NOT NULL COMMENT '表名，如 pixel_orders_table',
    last_success_time DATETIME     NOT NULL COMMENT '最近一次成功写入的业务时间',
    run_mode      VARCHAR(16)  NOT NULL COMMENT 'full=全量, incremental=增量',
    updated_at    DATETIME     NOT NULL COMMENT '水位线记录更新时间（UTC）'
)
UNIQUE KEY(source, table_name)
DISTRIBUTED BY HASH(source) BUCKETS 4
PROPERTIES (
    "replication_num" = "1"
);
```

**关键说明：**
- Unique Key 为 `(source, table_name)`，同 source+table 只有一条记录
- `ensure_table()` 使用 `CREATE TABLE IF NOT EXISTS`，幂等安全
- **Story 6.3 会扩展本表**（新增 `chunk_start`、`chunk_end`、`chunk_status` 字段），本 Story 不预留这些字段

**`ensure_table()` 实现要点：**

```python
def ensure_table() -> None:
    config = DorisConfig()
    conn = None
    ddl = """CREATE TABLE IF NOT EXISTS hqware.etl_watermark (...)"""
    try:
        conn = pymysql.connect(**config.DB_CONFIG)
        with conn.cursor() as cursor:
            cursor.execute(ddl)
        conn.commit()
        logger.info("[watermark] etl_watermark 表确认存在")
    except Exception as e:
        raise RuntimeError(f"[watermark] 建表失败：{e}") from e
    finally:
        if conn:
            conn.close()
```

**`get_watermark()` 实现要点：**

```python
def get_watermark(source: str, table: str) -> Optional[dict]:
    config = DorisConfig()
    conn = None
    try:
        conn = pymysql.connect(**config.DB_CONFIG)
        with conn.cursor(pymysql.cursors.DictCursor) as cursor:
            cursor.execute(
                "SELECT * FROM hqware.etl_watermark WHERE source=%s AND table_name=%s",
                (source, table)
            )
            row = cursor.fetchone()
        return row  # None if not found
    except Exception as e:
        raise RuntimeError(f"[watermark][{source}/{table}] 查询失败：{e}") from e
    finally:
        if conn:
            conn.close()
```

**`update_watermark()` 实现要点：**

```python
def update_watermark(source: str, table: str, success_time: datetime) -> None:
    existing = get_watermark(source, table)
    run_mode = "incremental" if existing else "full"
    record = {
        "source": source,
        "table_name": table,
        "last_success_time": success_time,
        "run_mode": run_mode,
        "updated_at": datetime.utcnow(),
    }
    write_to_doris(
        table=WATERMARK_TABLE,
        records=[record],
        unique_keys=["source", "table_name"],
        source=WATERMARK_SOURCE,
    )
    logger.info(f"[watermark][{source}/{table}] 水位线更新至 {success_time}，模式={run_mode}")
```

**`reset_watermark()` 实现要点：**

```python
def reset_watermark(source: str, table: str) -> None:
    config = DorisConfig()
    conn = None
    try:
        conn = pymysql.connect(**config.DB_CONFIG)
        with conn.cursor() as cursor:
            cursor.execute(
                "DELETE FROM hqware.etl_watermark WHERE source=%s AND table_name=%s",
                (source, table)
            )
        conn.commit()
        logger.info(f"[watermark][{source}/{table}] 水位线已重置（下次将全量拉取）")
    except Exception as e:
        raise RuntimeError(f"[watermark][{source}/{table}] 重置失败：{e}") from e
    finally:
        if conn:
            conn.close()
```
