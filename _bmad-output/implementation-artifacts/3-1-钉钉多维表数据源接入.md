# Story 3.1: 钉钉多维表数据源接入

Status: done

## Story

作为操作者，
我希望验证器能通过钉钉 OAuth2 App Key/Secret 连接多维表并抓取指定工作簿的字段数据，
以便我能获得钉钉多维表的实际字段报告。

## Acceptance Criteria

1. **Given** `get_credentials()` 返回有效的 `DINGTALK_APP_KEY` 和 `DINGTALK_APP_SECRET`，**When** 调用 `dingtalk.authenticate()`，**Then** 获取 access token 成功，返回 `True`，日志输出 `[dingtalk] 认证 ... 成功`

2. **Given** authenticate 成功后，**When** 调用 `dingtalk.fetch_sample()`，**Then** 通过钉钉多维表 API 返回至少一条记录，超时设置为 30s，`table_name` 参数为 `None`（非 SQL 数据源）

3. **Given** fetch_sample 返回的样本数据，**When** 调用 `dingtalk.extract_fields(sample)`，**Then** 返回符合标准 FieldInfo 结构的字段列表

4. **Given** API 认证失败，**When** 调用 `authenticate()`，**Then** 日志输出 `[dingtalk] 认证 ... 失败：{错误信息}`，返回 `False`

5. **Given** 单元测试环境（mock get_credentials + fixture），**When** 运行 `tests/test_dingtalk.py`，**Then** 所有单元测试通过，不需要真实凭证或网络访问

## Dev Notes

### 关键约束

**1. 仅使用 `requests` 库**

**2. 凭证通过 `import config.credentials as _creds_module` 获取**

**3. 日志格式遵守项目统一格式**：`[dingtalk] 认证 ... 成功`

**4. FieldInfo 标准四键结构**：`field_name / data_type / sample_value / nullable`

**5. token 缓存**：模块级 `_cached_token` + `_token_expiry`，过期前 60s 刷新

**6. HTTP 超时 30s**

## Code Review Findings

### F1: token 缓存未考虑并发安全 — 已延后
- 架构设计决策：单线程 CLI 场景，模块级全局变量无线程安全问题
- 延后至：架构评审或多线程需求出现时处理

### F2: `fetch_sample` 中 `creds.get("DINGTALK_SHEET_ID", "")` 二次调用 `get_credentials()`
- 延后修复：逻辑正确，不影响功能，性能优化留后处理

### F3: `fetch_sample` 未处理空响应（values 非列表情形）
- 延后修复：防御性编码，规范未要求，traceback 可见

### F4: 凭证导入使用 `import config.credentials as _creds_module` — 已确认为项目标准
- 与 `tiktok.py` 等其他模块对齐，无需修改
