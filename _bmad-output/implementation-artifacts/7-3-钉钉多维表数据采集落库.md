# Story 7.3：钉钉多维表数据采集落库

Status: done

## Story

作为数据工程师，
我希望能将钉钉多维表（Bitable）中的业务数据采集并写入 Doris，
以便集中存储 KOL 信息、合作价格等内部管理数据。

## Acceptance Criteria

1. 调用 `sdk/dingtalk/` 客户端（Story 6.0 已完成），不重复实现 token 管理逻辑
2. 支持已验证的全部 8 张多维表（见 TABLE_CONFIG），每张独立采集写入对应 Doris 表
3. 增量策略：全量拉取 + upsert（钉钉 API 无时间过滤接口，每次全量覆盖）
4. 关联字段（`linkedRecordIds` 结构）返回时，Doris 对应字段写 NULL，不报错
5. 写入策略：upsert，以钉钉行 `record_id` 为主键，幂等写入
6. 每张表采集完成后更新水位线（`update_watermark`），失败时不更新
7. 单表失败不中断其他表继续采集
8. 成功写入后日志格式：`[dingtalk_collector][hqware.xxx] 写入 N 行 ... 成功`
9. 单元测试覆盖：transform 函数 + collect 正常路径（mock SDK + mock write_to_doris）

### `_sanitize_record` 转换规则

```python
import re
from datetime import datetime, timezone

_MS_EPOCH_THRESHOLD = 1e12  # 毫秒级 epoch 均 > 1e12

def _is_linked_record(v: object) -> bool:
    return isinstance(v, dict) and "linkedRecordIds" in v

def _is_attachment(v: object) -> bool:
    return isinstance(v, list) and bool(v) and isinstance(v[0], dict) and (
        "url" in v[0] or "resourceUrl" in v[0]
    )

def _is_ms_timestamp(v: object) -> bool:
    return isinstance(v, (int, float)) and abs(v) > _MS_EPOCH_THRESHOLD

def _sanitize_record(raw: dict) -> dict:
    """将 API 原始记录转换为 Doris 可写入格式。"""
    result: dict = {}
    for k, v in raw.items():
        if k == "_record_id":
            result["record_id"] = v           # 重命名为 record_id（主键）
        elif _is_linked_record(v):
            result[k] = None                  # 关联记录 → NULL
        elif _is_attachment(v):
            result[k] = v[0].get("url") or v[0].get("resourceUrl") or None  # 取第一附件 URL
        elif _is_ms_timestamp(v):
            result[k] = datetime.fromtimestamp(v / 1000, tz=timezone.utc).replace(tzinfo=None)  # UTC datetime
        else:
            result[k] = v
    return result
```
