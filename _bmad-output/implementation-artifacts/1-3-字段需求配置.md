## Story

作为操作者，
我希望通过 YAML 配置文件按报表分组定义各数据源的字段需求清单，
以便在不修改任何代码的情况下更新字段需求。

## Acceptance Criteria

1. **AC1 — YAML 结构合法**：`config/field_requirements.yaml` 包含 `profit_table` 和 `marketing_table` 两个分组，使用 PyYAML 加载后返回字典，每条记录均包含 `display_name`（中文字段名）、`source`（数据源名称）、`table`（表/端点名称）三个字段。

### YAML 数据结构规范

```yaml
# config/field_requirements.yaml
# 按报表分组，每条记录含三个字段：
#   display_name: 中文需求字段名（唯一显示标识）
#   source: 数据源名称（对应 sources/ 目录下模块文件名，不含 .py）
#   table: 表/端点名称（非 SQL 数据源可为 null 或省略）

profit_table:           # 利润表
  - display_name: 日期
    source: triplewhale
    table: pixel_orders_table
  - display_name: 销售额
    source: triplewhale
    table: pixel_orders_table
  - display_name: SKU
    source: tiktok
    table: orders

marketing_table:        # 营销表现表
  - display_name: 曝光量
    source: triplewhale
    table: pixel_joined_tvf

youtube_table:          # 非 SQL 数据源示例（table 为 null）
  - display_name: 视频标题
    source: youtube
    table: null         # 或省略该键，PyYAML 均解析为 None
```

### reporter.py 如何消费此 YAML（来自 Story 1-4 设计）

```python
# reporter.py 中的加载模式（本 Story 无需实现，仅供参考）
import yaml
from pathlib import Path

def load_field_requirements() -> dict:
    path = Path("config/field_requirements.yaml")
    with open(path, encoding="utf-8") as f:
        return yaml.safe_load(f)

# 返回结构示例：
# {
#   "profit_table": [
#     {"display_name": "日期", "source": "triplewhale", "table": "pixel_orders_table"},
#     ...
#   ],
#   "marketing_table": [...],
# }
```

消费逻辑：`reporter.py` 会根据 `source` 字段过滤出当前数据源的所有需求字段，渲染到 Raw 报告的"需求字段（待人工对照）"区块。

### 测试实现指引

```python
# tests/test_field_requirements.py
import yaml
from pathlib import Path

YAML_PATH = Path("config/field_requirements.yaml")

def test_yaml_loads_successfully():
    """YAML 文件可正常加载，返回非空字典"""
    data = yaml.safe_load(YAML_PATH.read_text(encoding="utf-8"))
    assert isinstance(data, dict)
    assert len(data) > 0

def test_required_report_groups_exist():
    """profit_table 和 marketing_table 两个分组必须存在"""
    data = yaml.safe_load(YAML_PATH.read_text(encoding="utf-8"))
    assert "profit_table" in data
    assert "marketing_table" in data

def test_profit_table_has_minimum_entries():
    """profit_table 至少 3 条条目"""
    data = yaml.safe_load(YAML_PATH.read_text(encoding="utf-8"))
    assert len(data["profit_table"]) >= 3

def test_each_entry_has_required_fields():
    """每条记录必须含 display_name 和 source；table 可选"""
    data = yaml.safe_load(YAML_PATH.read_text(encoding="utf-8"))
    for group_name, entries in data.items():
        for entry in entries:
            assert "display_name" in entry, f"{group_name} 条目缺少 display_name"
            assert "source" in entry, f"{group_name} 条目缺少 source"
            # table 是可选字段，存在时可以为 None

def test_non_sql_source_table_optional():
    """非 SQL 数据源（youtube）的 table 字段为 None 或缺省不报错"""
    data = yaml.safe_load(YAML_PATH.read_text(encoding="utf-8"))
    # 找到 source == "youtube" 的条目（若存在）
    youtube_entries = [
        entry
        for entries in data.values()
        for entry in entries
        if entry.get("source") == "youtube"
    ]
    for entry in youtube_entries:
        # table 可为 None 或不存在，不应报错
        table_value = entry.get("table")
        assert table_value is None or isinstance(table_value, str)
```

> **注意**：测试直接读取 `config/field_requirements.yaml` 文件路径，运行时需在项目根目录执行 `pytest`。无需 mock `get_credentials()`（本 Story 不涉及凭证）。
