# Story 1.4: 报告渲染器

Status: done

## Story

作为操作者，
我希望有一个报告渲染器能生成 raw 字段发现报告和 validation 人工标注模板，
以便我能看到实际 API 返回字段与需求字段的对照视图，并在 validation 文件中填写人工判断结论。

## Acceptance Criteria

1. **Given** 一个数据源名称、FieldInfo 列表、表名和样本数量；**When** 调用 `write_raw_report(source_name, fields, table_name, sample_count)`；**Then** `reports/{source_name}-raw.md` 被创建（或覆盖），包含：生成时间戳、表名、样本记录数、实际返回字段表格（字段名/类型/示例值/可空）

2. **Given** `config/field_requirements.yaml` 已加载；**When** `write_raw_report()` 被调用；**Then** 报告末尾包含"需求字段（待人工对照）"区块，列出该数据源所有 `display_name` 字段，初始状态标记为 `⬜ 待确认`

3. **Given** `reports/{source_name}-validation.md` 不存在；**When** 调用 `init_validation_report(source_name)`；**Then** 创建人工标注模板文件，包含三态标注占位符（✅ 直接可用 / ⚠️ 需转换 / ❌ 缺失）、转换逻辑说明行、替代方案说明行

4. **Given** `reports/{source_name}-validation.md` 已存在且有人工标注内容；**When** 再次调用 `init_validation_report(source_name)`；**Then** 现有文件**不被覆盖**，人工标注内容完整保留（通过 `Path.exists()` 判断）

5. **Given** raw 报告生成完成；**When** 检查报告内容；**Then** 不包含任何完整 API Key、Token 或密码值

### FieldInfo 标准结构（输入参数 fields 的格式）

```python
# 每个 FieldInfo 字典固定四字段，任何 source 模块均遵守此结构

    "field_name": str,      # 字段名（API 返回的原始名称）
    "data_type": str,       # 类型：string / number / boolean / array / object / null
    "sample_value": any,    # 示例值（脱敏后，不含凭证信息）
    "nullable": bool        # 是否可为空
}
```

### raw 报告 Markdown 格式（严格遵守）

```markdown
# {Source} 字段验证报告（Raw）

**生成时间：** {datetime}
**数据表：** {table_name}
**样本记录数：** {count}

## 实际返回字段

| 字段名 | 类型 | 示例值 | 可空 |
|--------|------|--------|------|
| {field_name} | {type} | {sample} | {nullable} |

## 需求字段（待人工对照）

| 需求字段（中文） | 报表 | 对照结果 |
|----------------|------|---------|
| {display_name} | {report} | ⬜ 待确认 |
```

- `table_name` 为 `None` 时（非 SQL 数据源如 YouTube），显示为 `N/A`
- 时间戳格式：`datetime.now().strftime("%Y-%m-%d %H:%M:%S")`
- 报告写入路径：`Path("reports") / f"{source_name}-raw.md"`

### validation 报告 Markdown 格式

```markdown
# {Source} 字段对标结论（Validation）

> 本文件由工具首次生成，后续**人工维护**，工具运行不覆盖。
> 创建时间：{datetime}

## 字段对标结论

| 需求字段（中文） | 报表 | 状态 | 说明 |
|----------------|------|------|------|
| {display_name} | {report} | ⬜ 待确认 | |

---

### 标注说明

- ✅ 直接可用：API 实际返回该字段，可直接使用
- ⚠️ 需转换：有对应字段但需格式转换，在"说明"列填写转换逻辑
- ❌ 缺失：API 无此字段，在"说明"列填写替代方案或结论
- ⬜ 待确认：尚未完成对标判断
```

### field_requirements.yaml 加载方式

```python
# 路径：config/field_requirements.yaml
# 注意：架构文档使用 field_requirements.yaml（下划线命名）
REQUIREMENTS_PATH = Path("config") / "field_requirements.yaml"

def _load_field_requirements() -> dict:
    """加载字段需求配置，返回完整 dict。文件不存在时返回空 dict。"""
    if not REQUIREMENTS_PATH.exists():
        return {}
    with open(REQUIREMENTS_PATH, encoding="utf-8") as f:
        return yaml.safe_load(f) or {}
```

**过滤逻辑**：报告渲染时，遍历所有 report_group（如 `profit_table`、`marketing_table`），筛选 `item["source"] == source_name` 的条目作为该数据源的需求字段。

```python
# 示例：从 YAML 中提取 triplewhale 数据源的所有需求字段
req = _load_field_requirements()
source_fields = []
for report_name, items in req.items():
    if isinstance(items, list):
        for item in items:
            if item.get("source") == source_name:
                source_fields.append({
                    "display_name": item["display_name"],
                    "report": report_name,
                    "table": item.get("table")
                })
```

### validation.md 保护逻辑（关键）

```python
def init_validation_report(source_name: str) -> None:
    _ensure_reports_dir()
    path = REPORTS_DIR / f"{source_name}-validation.md"
    if path.exists():
        logger.info(f"[reporter] {source_name} validation 报告已存在，跳过创建")
        return  # 关键：不覆盖现有文件
    # 仅首次创建模板
    content = _render_validation_template(source_name)
    path.write_text(content, encoding="utf-8")
    logger.info(f"[reporter] {source_name} validation 报告模板已创建")
```

### 凭证安全（AC 5）

reporter.py **本身不接触凭证**，但 `sample_value` 可能包含敏感信息（如 Token、密码）。在 `_format_sample_value()` 辅助函数中，不做额外过滤——脱敏职责在各 source 模块的 `extract_fields()` 中完成。reporter.py 只渲染传入的 FieldInfo 数据。
